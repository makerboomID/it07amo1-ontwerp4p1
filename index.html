Ontwerp 4 - Git-flow

Introductie

Doelstelling

  • Behendig zijn in: commits, branching, merging, pushing en pulling
  • Merge conflicten kunnen oplossen
  • Kunnen werken volgens de gitflow methode

Startinstructies

  • Fork deze repository naar jouw eigen github account
  • Clone deze fork naar jouw computer
  • Open je favoriete git applicatie en editor.

Afbeeldingen toevoegen

Opdracht

Voor je begint met het aanpassen van de code, zorg dat je in de develop branch zit.

  • Maak een nieuwe branch: feature/afbeeldingen
  • Controleer dat je in deze branch zit
  • Herhaal de volgende stappen voor iedere hoofdstuk.
    • Zoek een passende afbeelding voor het hoofdstuk, voeg deze afbeelding toe in de map images en maak een commit. zoeken
    • Pas de src aan van de placeholder afbeelding van het introductie hoofdstuk aan, zodat de nieuw toegevoegde afbeelding zichtbaar is en maak een commit.
  • Ga terug naar de develop branch.
  • Merge de aanpassingen van feature/afbeeldigen naar develop.
  • Push de develop branch naar github

Een conflict oplossen

Winnaar: Onbekend

Chuck

  • Begin een nieuwe feature branch: feature/chuck
  • Pas de hierboven aangegeven winnaar aan naar: Chuck Norris
  • Commit deze wijziging.

Superman

  • Ga terug naar de develop branch
  • Begin een nieuwe feature branch: feature/superman
  • Pas de hierboven aangegeven winnaar aan naar: Superman
  • Commit deze wijziging.

Fight!

  • Ga terug naar de develop branch
  • Merge de aanpassingen van feature/superman naar develop.
  • Controleer of superman de winnaar is.
  • Merge de aanpassingen van feature/chuck naar develop.
  • Conflict! 
    
                                <<<<<<< HEAD
                                <span id="winnaar">Superman</span>
                                =======
                                <span id="winnaar">Chuck</span>
                                >>>>>>> feature/chuck
  • Zorg dat Chuck de winnaar is.
  • Commit de merge
  • Push de develop branch naar github

Release

  • Ga naar de develop branch.
  • Start een nieuwe release branch: release/{versie}
  • Pas de versie aan in de index.html
    Boven aan de index.html, zie je de laatse versie, jouw {versie} is 1 cijfer hoger.
  • Commit deze aanpassing
  • Ga naar de develop branch
  • Merge de aanpassingen van release/{versie} naar develop.
  • Push de develop branch naar github
  • Ga naar de master branch
  • Merge de aanpassingen van release/{versie} naar master.
  • Push de master branch naar github
